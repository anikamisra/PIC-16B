First, let’s create a function to clean up the temperatures dataframe a bit.

def prepare_df(df):
    """
    Function that cleans and reorganizes the temperatures dataframe. 
    
    args: 
        df (pandas.DataFrame): the dataframe to organize, from the temperatures file of the NOAA database 
    returns: 
        df (pandas.DataFrame): the reorganized dataframe 
    """
    
    df = df.set_index(keys=["ID", "Year"])
    df = df.stack()
    df = df.reset_index()
    df = df.rename(columns = {"level_2"  : "Month" , 0 : "Temp"})
    df["Month"] = df["Month"].str[5:].astype(int)
    df["Temp"]  = df["Temp"] / 100
    return(df)

Now let’s add the temperatures dataframe to our SQL database as a table by working with only 100,000 rows at a time. We are able to do this thanks to the iterator we created.

for i, df in enumerate(temps_df_iter):

Part 2: Querying our first database

Let’s now use our database to perform an SQL query that returns a pandas dataframe. The beauty of this is that we do not have to have the entire dataframe in our RAM. To do this, we will use the query_climate_database() function which I have already defined. Let’s see what it looks like below:

from climate_database import query_climate_database
import inspect
print(inspect.getsource(query_climate_database))
def query_climate_database(db_file, country, year_begin, year_end, month): 
    """
    Function which performs SQL query based on country, year beginning, year end, and month. 
    
    args: 
        db_file (str): string for database name 
        country (str): string for country 
        year_begin (int): year in XXXX format for which year you want to start from 
        year_end (int): year in XXXX format for which year you want to end from 
        month (int): integer value of the month you are trying to query 
    returns: 
        df (pandas.DataFrame): a dataframe with these selected values 
    """
    
    conn = sqlite3.connect(db_file) 
    

    query = f"""
    SELECT 
        temperatures.Year, 
        temperatures.Month, 
        temperatures.Temp, 
        stations.NAME, 
        stations.LATITUDE, 
        stations.LONGITUDE, 
        countries.Name 
    FROM 
        temperatures
    JOIN 
        stations ON temperatures.ID = stations.ID
    JOIN 
        countries ON substr(temperatures.ID, 1, 2) = countries.[FIPS 10-4]
    WHERE 
        countries.Name = '{country}'
        AND temperatures.Year BETWEEN {year_begin} AND {year_end}
        AND temperatures.Month = {month}
    """
    
    df = pd.read_sql_query(query, conn)
    df = df.reindex(columns=['NAME', 'LATITUDE', 'LONGITUDE', 'Name', 'Year', 'Month', 'Temp'])
    df.columns = ['NAME', 'LATITUDE', 'LONGITUDE', 'Country', 'Year', 'Month', 'Temp']
    
    conn.close()
    
    return df

So this function takes 5 arguments 1. Database name 2. Country name 3. Beginning year 4. Ending year 5. Month number

Part 3: Geographic scatterplots

Now, let’s use SQL querying to create some geographic scatterplots with the NOAA data. First, let’s take a look at the function which will plot this figure for us:

from climate_database import temperature_coefficient_plot
print(inspect.getsource(temperature_coefficient_plot))
def temperature_coefficient_plot(db_file, country, year_begin, year_end, month, min_obs, **kwargs):
    """
     Function which creates a geographic scatterplot to see how yearly temperature increase vary across different locations (stations) for a given timeframe for a certain country. 
    
    args: 
        db_file (str): string for database name 
        country (str): string for country to be plotted 
        year_begin (int): year in XXXX format for which year you want to start from 
        year_end (int): year in XXXX format for which year you want to end from
        month (int): integer value of the month you are trying to query
        min_obs (int): the minimum observations per data point plotted 
        **kwargs: arbitrary keyword arguments for the plot 
        
    returns: 
        fig (plotly.express.scatter_mapbox): A geographic scatterplot with yearly temperature increase at different stations.
    """
    # Query the climate database
    df = query_climate_database(db_file, country, year_begin, year_end, month)
    # use transform function here to filter out the stations that don't have at least min_obs years worth of data 
    
    # Group by station and calculate the yearly change in temperature
    df = df[df.groupby('NAME')['Year'].transform('count') >= min_obs] 
    
    coefs = df.groupby('NAME').apply(coef).reset_index()
    coefs.columns = ['NAME', 'YearlyChange']
    
    df = pd.merge(df, coefs, on='NAME')
    
    #df['YearlyChange'] = df['YearlyChange'].round(4)
    
    # Create the scatterplot
    fig = px.scatter_mapbox(df, lat='LATITUDE', lon='LONGITUDE', color='YearlyChange',
                            color_continuous_midpoint=0, # centers the colorbar at 0 
                            hover_data={'NAME': True, 'YearlyChange': ':.3f'}, # shows 3 digits after the decimal point 
                            labels={'YearlyChange': 'Estimated Yearly Increase (°C)'}, 
                            title=f"Estimates of yearly increase in temperature in {pd.to_datetime(month, format='%m').month_name()} for stations in {country}, years {year_begin}-{year_end}",  
                            **kwargs)
    
    return fig

As we can see, the temperature_coefficient_plot function takes in 6 or more arguments: 1. database name 2. country name 3. beginning year 4. end year 5. month number 6. minimum observations in data point 7. additional keyword arguments for graph

It uses the first 5 arguments as input in the query_climate_database we tested earlier. Using this information, let’s create a geographic scatterplot of the yearly increase in temperature in January for India from 1980 to 2020.

import plotly.express as px

color_map = px.colors.diverging.RdGy_r # choose a colormap

                                   min_obs = 10,
                                   zoom = 2,
                                   mapbox_style="carto-positron",
                                   color_continuous_scale=color_map)

fig.show()

Interesting! The estimated yearly increase actually appears to be decreasing near the Northeast region next to the Himalayas. I guess that makes sense, seeing how cold those mountains are. Let’s use the same function to see how this compares to Russia.

Part 4: Other types of Interactive Plots

Let’s create a new query function that does not depend on the month. We can use this to create other kinds of interactive plots with plotly, that will answer two important questions:

Question 1: How does the average yearly change in temperature vary based on elevation in a given country for a given timeframe?

Question 2: How does the month impact the yearly increase in temperature for certain years in a given country?

Let’s get started. First, let’s take a look at what the new query function looks like:

from climate_database import query_climate_database_2
print(inspect.getsource(query_climate_database_2))
def query_climate_database_2(db_file, country, year_begin, year_end): 
    """
    Function which performs SQL query based on country, year beginning, and year end. 
    
    args: 
        db_file (str): string for database name 
        country (str): string for country 
        year_begin (int): year in XXXX format for which year you want to start from 
        year_end (int): year in XXXX format for which year you want to end from 
    returns: 
        df (pandas.DataFrame): a dataframe with these selected values 
    """
    conn = sqlite3.connect(db_file) 
    
    query = f"""
    SELECT 
        temperatures.Year, 
        temperatures.Month,  
        temperatures.Temp, 
        stations.NAME, 
        stations.LATITUDE, 
        stations.LONGITUDE, 
        stations.STNELEV,
        countries.Name
    FROM 
        temperatures 
    JOIN 
        stations ON temperatures.ID = stations.ID
    JOIN 
        countries ON substr(temperatures.ID, 1, 2) = countries.[FIPS 10-4]
    WHERE 
        countries.Name = '{country}'
        AND temperatures.Year BETWEEN {year_begin} AND {year_end}
    """
    df = pd.read_sql_query(query, conn) 
    conn.close() 
    return df

As we can see, this query function is the same as the previous one except it does not take in month. We can use this query function to create pandas dataframes that are more generalized, and then create plots to let us see more longterm trends. Let’s start with question 1, which is simple to plot but interesting to know.

Question 1: How does the average yearly change in temperature vary based on elevation in a given country for a given timeframe?

To answer this question, we will use the elevation_coefficient_plot() function. Let’s inspect this function to see what it looks like:

from climate_database import elevation_coefficient_plot
print(inspect.getsource(elevation_coefficient_plot))
def elevation_coefficient_plot(db_file, country, year_begin, year_end, **kwargs): 
    """
     Function which creates an interactive plot for elevation vs. yearly increase in temperature for a given timeframe for a certain country. 
    
    args: 
        db_file (str): string for database name 
        country (str): string for country to be plotted 
        year_begin (int): year in XXXX format for which year you want to start from 
        year_end (int): year in XXXX format for which year you want to end from
        **kwargs: arbitrary keyword arguments for the plot 
        
    returns: 
        fig (plotly.express.scatter): A scatter plot of elevation on the x-variable and yearly change on the y-variable.  
    """
    
    df = query_climate_database_2(db_file, country, year_begin, year_end)
    
    coefs = df.groupby('NAME').apply(coef).reset_index()
    coefs.columns = ['NAME', 'YearlyChange']
    
    df = pd.merge(df, coefs, on='NAME')
    
    fig = px.scatter(df, x='STNELEV',
                     y='YearlyChange',
                     hover_data={'NAME': True, 'LATITUDE': ':.3f', 'LONGITUDE': ':.3f', 'YearlyChange': ':.3f'},
                     labels={'STNELEV': 'Station Elevation (meters)', 'YearlyChange': 'Yearly Temperature Change (°C)'},
                     title=f'Correlation between Elevation and Yearly Increase in Temperature in {country} for the years {year_begin}-{year_end}', 
                     **kwargs)
    return fig

As we can see, this function takes in the same 4 arguments as query_climate_database_2, except with room for one more optional argument for additional keyword arguments. The way this function works is: 1. It first performs the query_climate_database_2 function to obtain the necessary data. 2. Then, it creates a dataframe for this data. 3. Finally, creates an interactive plotly scatterplot that allows us to see the correlation between elevation and average yearly change in temperature.

Let’s see if we can find any patterns for India from 1980 to 2020.

fig = elevation_coefficient_plot("database.db", "India", 1980, 2020)
fig.show()

Wow! As we can see, each value is nicely rounded, and we can hover over each datapoint to learn more about it, such as the station name, the exact yearly temperature change, and more. However, there does not appear to be much correlation at all. So, to answer our question:

In India from 1980-2020, the average yearly change in temperature does not depend on elevation.

But, we can always make more scatterplots to see if there is any correlation in other countries!

However, while it is informative, this type of plot is not extremely exciting. Let’s create another plot that has more facets. We now want to answer the question:

Question 2: How does the month impact the yearly increase in temperature for certain years in a given country?

To do this, we will use the same query_climate_database_2 function, but this time, use the temperature_increase_boxplot function to graph it. Let’s, once again, take a look at what this function looks like.

from climate_database import temperature_increase_boxplot
print(inspect.getsource(temperature_increase_boxplot))
def temperature_increase_boxplot(db_file, country, year_begin, year_end, **kwargs): 
    """
     Function which creates an interactive boxplot to see how yearly temperature increase varies across the months, for a given timeframe for a certain country. 
    
    args: 
        db_file (str): string for database name 
        country (str): string for country to be plotted 
        year_begin (int): year in XXXX format for which year you want to start from 
        year_end (int): year in XXXX format for which year you want to end from
        **kwargs: arbitrary keyword arguments for the plot 
        
    returns: 
        fig (plotly.express.box): A boxplot of yearly temperature increase over the months.
    """
    df = query_climate_database_2(db_file, country, year_begin, year_end)

    
    coefs = df.groupby(['NAME', 'Month']).apply(coef).reset_index()
    coefs.columns = ['NAME', 'Month', 'YearlyChange']
    
    df = pd.merge(df, coefs, on=['NAME', 'Month'])
    df['Month'] = pd.to_datetime(df['Month'], format='%m').dt.month_name()
    df['YearlyChange'] = df['YearlyChange'].round(3)
    
    # Create the boxplot
    fig = px.box(df, x='Month', y='YearlyChange',
                 color = 'Month',
                 labels={'Month': 'Month', 'YearlyChange': 'Yearly Temperature Change (°C)'},
                 title=f'Month-by-Month Statistics of Yearly Increase in Temperature in {country} from {year_begin}-{year_end}',
                 hover_data={'YearlyChange': ':.3f'},
                 **kwargs)
    
    return fig

temperature_increase_boxplot takes in the same arguments as elevation_coefficient_plot, except this time, it makes a box plot based on every month of the year. Let’s see this in action.

fig = temperature_increase_boxplot("database.db", "India", 1980, 2020)

fig.show()

Wow, what an informative plot! The axes are nicely labelled, and, if we look closely, we can see a monthly pattern in yearly increase in temperature. It appears to be going in a slight wave up and down. However, the data points do have a lot of outliers.
